Zero-Trust Photographic Verification

Trust in Verification.

A 100% serverless, database-free web application. It guarantees that a photo is taken live, unedited, and unaltered through cryptographically secure asymmetric Curve25519 encryption.

View GitHub Repo
📷

Mandatory Live Camera

Strictly enforces hardware camera capture. The system rejects file uploads, ensuring the image is captured in real-time at the exact moment of verification.

🔐

Asymmetric Encryption

Powered by Tweetnacl.js (Curve25519). The sender cannot view or edit the photo after capture. Only the requester, holding the local private key, can decrypt it.

☁️

Zero Server Storage

No backend databases. No cloud storage. Everything happens securely in your browser's memory, ensuring your most sensitive verifications remain entirely private.

The Lock & Key Protocol

How we mathematically guarantee authenticity.

01

Requester Generates Keypair

The owner creates a secure link. A private key is generated and stored locally in their browser's IndexedDB.

02

Sender Captures & Encrypts

The sender opens the link, takes a live photo, and the app instantly encrypts it with the public key. The image is never saved raw.

03

Distribution

The resulting opaque `.capture` file is shared via WhatsApp or Email. It is useless without the private key.

04

Owner Unlocks

The requester uploads the `.capture` file back into the app, which uses the local private key to reveal the verified image.
